5. Sebuah alamat Ip sebelum di subneting adalah 192.68.2.1, jika di sebuah kantor ingin membagi ke dalam 8 network baru maka berapakah subnet-nya dan berapa ip networknya tiap subnet !


Jawaban:

Alamat IP 192.68.2.1 dengan subnet 3 bit akan menghasilkan 8 subnet baru dengan subnet mask 255.255.255.224 dan tiap subnet dapat menampung 254 alamat IP. Tiap subnet memiliki alamat IP dari network address hingga broadcast address.

Penjelasan:

Untuk membagi alamat IP 192.68.2.1 menjadi 8 network baru, kita perlu menggunakan proses subnetting. Dalam subnetting, kita membagi alamat IP menjadi beberapa subnet dengan mengubah bagian dari bit host menjadi bit subnet.

Langkah-langkah untuk menghitung subnet dan alamat IP tiap subnet adalah sebagai berikut:

Tentukan jumlah subnet yang dibutuhkan, dalam kasus ini adalah 8.

Tentukan jumlah bit subnet yang diperlukan dengan menggunakan rumus 2^b >= jumlah subnet, dimana b adalah jumlah bit subnet yang diperlukan. Dalam kasus ini, 2^3 >= 8, maka diperlukan 3 bit subnet.

Hitung nilai dari setiap bit subnet. Dalam kasus ini, bit subnet adalah 3 bit paling kiri dari octet ke-3. Maka nilai dari bit subnet adalah 128 + 64 + 32 = 224.

Hitung jumlah alamat IP dalam setiap subnet dengan menggunakan rumus 2^(32-b)-2, dimana b adalah jumlah bit subnet yang diperlukan. Dalam kasus ini, 2^(32-3)-2 = 254, artinya setiap subnet dapat menampung 254 alamat IP.

Bagi alamat IP menjadi 8 subnet dengan menambahkan nilai bit subnet ke setiap subnet. Untuk menghitung alamat IP tiap subnet, gunakan rumus subnet mask = 256 - nilai bit subnet.

9. Jika diberikan IP Address 192.168.170.155/27 lakukan subnetting ! Berapa ? 1. IP BC (IP Awal) 2. IP Net (IP Akhir) 3. IP Host (rentang IP) 4.Subnet Mask 5.Jumlah Subnet 6. Blok Subnet 7. Host per subnet dan beserta caranya​


Jawaban:

IP Address 192.168.170.155/27 adalah alamat IP dengan subnet mask /27, yang menunjukkan bahwa 27 bit dari 32 bit alamat IP tersebut digunakan untuk menentukan subnet, sisanya 5 bit digunakan untuk menentukan host.

1. IP BC (IP Awal): 192.168.170.128

Untuk menentukan IP BC, pertama-tama kita harus menentukan subnet mask dalam bentuk desimal, yaitu 255.255.255.224. Kemudian, kita menentukan subnet dalam bentuk biner, yaitu 11000000.10101000.10101010.10000000. Setelah itu, kita membuat IP BC dengan mengubah hanya satu bit dari bit paling kanan dari subnet menjadi nol, sehingga memperoleh alamat IP dalam bentuk desimal sebagai berikut: 192.168.170.128.

2. IP Net (IP Akhir): 192.168.170.159

Untuk menentukan IP Net, kita melakukan hal yang sama dengan menentukan IP BC, tetapi dengan mengubah satu bit dari bit paling kanan dari subnet menjadi satu, sehingga memperoleh alamat IP dalam bentuk desimal sebagai berikut: 192.168.170.159.

3. IP Host (rentang IP): 192.168.170.129 - 192.168.170.154

IP Host adalah rentang IP yang dapat digunakan untuk menentukan host pada jaringan, dan dapat ditemukan dengan mengurangi IP BC dengan satu dan menambahkan IP Net dengan satu.

4. Subnet Mask: 255.255.255.224

Subnet Mask adalah alamat IP yang digunakan untuk menentukan bagian dari alamat IP yang digunakan untuk subnet.

5. Jumlah Subnet: 8

Jumlah subnet adalah jumlah subnet yang dapat dibuat dengan menggunakan alamat IP dan subnet mask tertentu. Dalam hal ini, karena kita memiliki 5 bit untuk menentukan host, kita memiliki 2^5 = 32 subnet yang dapat dibuat.

6. Blok Subnet: 32

Blok Subnet adalah jumlah alamat IP yang tersedia pada setiap subnet. Dalam hal ini, karena kita memiliki 2^5 = 32 subnet, maka blok subnet adalah 32.

7. Host per subnet: 30

Jumlah host per subnet adalah jumlah host yang dapat ditempatkan pada setiap subnet. Dalam hal ini, karena kita memiliki 2^5 - 2 = 30 host per subnet.

12. nana membeli sebuah flashdisk berkapasitas 16gb kapasitas flashdisk tersebut jika dinyatakan dalam byte adalah


Kapasitas flashdisk jika dinyatakan dalam byte adalah 16.000.000.000 byte

Pembahasan

AWALAN - AWALAN BESARAN

Di sekolah dasar kita diajari tangga satuan panjang dan satuan massa. Satuan panjang seperti km, hm, dam, m, dm, cm, mm. Satuan massa seperti kg, hg  dag, g, dg, cg, mg. Sebenarnya semua awalan yang dipakai adalah awalan - awalan dari sebuah besaran.

Dalam fisika, semua awalan ini bisa dipakai dalam besaran apapun untuk meringkas penggunaan angka nol yang terlalu panjang.

Awalan - awalan itu adalah

1. Tera (T) ⇒ 10¹² = 1.000.000.000.000

2. Giga (G) ⇒ 10⁹ = 1.000.000.000

3. Mega (M) ⇒ 10⁶ = 1.000.000

4. Kilo (k) ⇒ 10³ = 1.000

5. Hekto (h) ⇒ 10² = 100

6. Deka (da) ⇒ 10¹ = 10

7. Desi (d) ⇒ 10⁻¹ = 0,1

8. Centi (c) ⇒ 10⁻² = 0,01

9. Mili (m) ⇒ 10⁻³ = 0,001

10. Mikro (μ) ⇒ 10⁻⁶ = 0,000001

11. Nano (n) ⇒ 10⁻⁹ = 0,000000001

12. Piko (p) ⇒ 10⁻¹² = 0,000000000001

13. Femto (f) ⇒ 10⁻¹⁵ = 0,000000000000001

14. Atto (a) ⇒ 10⁻¹⁸ = 0,000000000000000001

Awalan - awalan ini dipergunakan di depan sebuah besaran.

Contoh

* 5.000 N = 5 × 10³ =  5 kN

* 0,0015 sekon = 1,5 × 0,001 s = 1,5 ms

* 0,0000000027 gram = 2,7 × 0,000000001 gr = 2,7 ngram

* 5.400.000.000 A = 5,4 × 1.000.000.000 A = 5,4 GA

Penulisan awalan tidak boleh sembarangan. Bila Mega harus ditulis dengan M kapital bukan m huruf kecil. Karena artinya akan berbeda.

Diket:

Sepertinya ada kesalahan penulisan seharusnya menggunakan huruf kapital.

Kapasitas = 16 GB

Dit:

Kapasitas dalam satuan byte?

Penjelasan:

16 GB = 16 × 10⁹ Byte

= 16 × 1.000.000.000 Byte

= 16.000.000.000 Byte

23. mengapa flashdisk berukuran 16GB menjadi kurang lebih 15GB, apa yang menyebabkan ?


Berdasarkan informasi yang saya dapatkan, hal tersebut disebabkan karena ada beberapa faktor, yaitu :
1. File SystemFile system yang ada dalam flashdisk atau hard disk tersebut. File system tersebut ada agar bisa dijalankan di komputer (semacam progrum autorun, dsb), namun kapasitas dari file tersebut tidak terlalu besar.
2.  Beda SatuanNah sebenarnya inilah biang keroknya kenapa pada flashdisk maupun hard disk kita kapasitasnya berkurang banyak meskipun baru dibeli. Secara teori satuan dalam komputer dan pemrosesan data dikenal dengan istilah Byte. Byte merupakan ukuran sebuah informasi. Tingkat satuan byte dari terendah dimulai Byte, KiloByte, MegaByte, GigaByte (selanjutnya TerraByte, PetaByte, EksaByte). Untuk lebih jelasnya bisa dilihat dibawah ini 

1 GigaByte (Gb)   : 1024 x 1024 x 1024 Byte1 MegaByte (Mb) : 1024 x 1024 Byte
1 KiloByte (Kb)     : 1024 Byte1 Byte (b)             : 1 Byte

Seperti yang anda liat, setiap turun satuan dikalikan dengan 1024 dan naik satuan dibagi dengan 1024. 
Terus yang menjadi pertanyaan adalah kenapa kapasitas flashdisk tidak sesuai dengan yang tertera di Box-nya?
Jawabanya adalah produsen Flashdisk maupun HardDisk tidak menggunakan satuan diatas. Mereka menganggap setiap turun maupun naik satuan dikalikan/dibagi dengan 1000. Loh kenapa ko begitu? ya biasalah pengusaha / pabriknya, selalu menginginkan keuntungan lebih.
Untuk lebih jelasnya saya berikan contoh :Pada saat kita membeli flashdisk ukuran 16 Gb maka secara hitungan pabrik yang sebenarnya akan ketemu nilai 16 x 1000 x 1000 x 1000 = 16.000.000.000 Byte
Namun pada saat kita mencolokkan ke komputer atau laptop maka kapasitasnya akan terbaca menjadi 16.000.000.000 Byte : 1024 : 1024 : 1024 = 14, 9 Gb. Nah dasar itulah yang menyebabkan ukuran / kapasitas Flashdisk maupun HardDisk tidak sesuai dengan yang tertera. Semakin besar Kapasitasnya maka semakin besar pula kapasitas Flashdisk maupun Hard Disk yang berkurang.
