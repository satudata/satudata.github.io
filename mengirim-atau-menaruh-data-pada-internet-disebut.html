6. Jelaskan cara bagaimana pengiriman data melalui internet!


Bit – bit data diterima oleh host penerimaLewat mana menerimanya? Tentu saja dari layer terbawah, yakni layer physical melalui NIC, dan kemudian NIC ini memprosesnya.

Bit – bit data diubah menjadi frameSelanjutnya. Bit – bit data tersebut diterima oleh layer Data Link. Tapi, sebelum dapat sepenuhnya diterima, Layer akan mengecek terlebih dahulu apakah MAC Adress yang ada di dalam frame sama dengan host penerima ini? Jika iya, frame ini akan masuk ke tahap selanjutnya, tapi jika tidak frame tersebut boleh dibuang.

Frame dibongkar menjadi paket data.Frame ini selanjutnya dibongkar. Apa sebenarnya frame itu? Frame disini diibaratkan sebagai wadah segmen yang juga dibungkus dengan aturan yang kita sebut protocol. Hasil dari pembongkaran ini adalah paket data.

Paket data diubah menjadi segmenProses ini menyisakan segmen untuk dapat diproses di layer di atasnya. Proses ini terjadi di layer network

Penyatuan SegmenProses ini terjadi pada layer transport. Apa yang terjadi? Segmen – segmen tersebut akan dicek protokolnya, dan akan diproses sesuai dengan jenis protokolnya. Kemudian, segmen – segmen ini akan disatukan kembali sesuai dengan nomor urutnya. Dan kemudian diangkut ke session layer untuk diatur waktu aliran datanya agar tidak tumpang tindih atau bertabrakan satu sama lain.

Penerjemahan dataIni adalah bagian dari tugas Layer Presentation. Data akan dicek apakah formatnya terkait dengan layer presentation dan kemudian diterjemahkan agar dapat dipahami oleh user yang berinteraksi langsung dengan layer aplikasi.

Tahap akhir, sampai tujuanData yang ada pada layer presentation itu selanjutnya kembali diangkut ke layer aplikasi setelah berhasil diterjemahkan. Dan user pun akhirnya dapat melihat data – data tersebut.

Informasi yang akan dikirimkan dikonversikan menjadi data.Proses ini terjadi pada layer aplikasi. 18. Cara kerja pengiriman data melalui jaringan internet


Cara kerja pengiriman data melalui internet, yaitu:

Data dalam format bit masih dikemas dalam paket data. Setiap paket data diberi label dengan informasi penting seperti alamat pengirim, alamat penerima, jenis paket, dan alamat proxy. Komputer pengirim pertama-tama menggunakan proses pencarian alamat untuk berkomunikasi dengan komputer penerima. Komputer pengirim mengirimkan tiga paket, paket TCP, paket ping ICMP, dan paket UDP, ke komputer penerima. Router memeriksa ping yang masuk dan meneruskan data ke alamat tujuan. Switch mencantumkan rute tercepat ke data dengan terlebih dahulu memeriksa integritas informasi dalam data. Setelah alamat tujuan ditentukan, data aktual dari komputer pengirim dipecah menjadi paket-paket yang lebih kecil yang dapat dilewatkan melalui jalur pengiriman sesuai dengan bandwidth yang diperiksa sebelumnya. Setiap paket data ditandai oleh switch dengan alamat IP tujuan.Setelah paket tiba di alamat tujuan, paket data direposisi ke format aslinya. Setiap alamat proxy di setiap paket diperiksa, dan jika konten alamat salah, konten dihapus, jika tidak paket diteruskan ke jalur Internet. Firewall memilih paket yang dapat dan tidak dapat melewati firewall. Paket yang tidak dapat melewati firewall dijatuhkan, dan paket yang dapat melewati firewall masuk ke intranet. Setelah paket melewati firewall, paket tersebut dijemput oleh router dan ditempatkan di lalu lintas jaringan, atau yang biasa disebut dengan bandwidth atau lalu lintas data.Pada tingkat berikutnya, paket memasuki dunia intranet. Setelah memasuki dunia intranet, paket tidak selalu aman. Ada ping di jalur lalu lintas jaringan. Ping of death adalah jenis serangan pada komputer yang melibatkan pengiriman ping yang salah. Ketika sebuah paket tiba di komputer tujuan, firewall di komputer itu melihat informasi port untuk setiap paket untuk menentukan port mana yang dikirim oleh setiap paket. Jika ada paket masuk pada port yang salah atau tidak ada, paket yang buruk akan dihapus. Bagian terakhir adalah memeriksa integritas paket. Jika data rusak, komputer penerima akan mengirimkan informasi yang rusak dan komputer pengirim akan mengirim kembali data yang rusak. Setelah selesai, komputer penerima menangkap data yang terkandung dalam setiap paket dan menyusunnya kembali menjadi informasi yang lengkap untuk dibaca oleh pengguna penerima.

Pembahasan

Pergerakan data membentang ribuan mil di seluruh pusat data Google. Data yang disimpan di server Google ditransmisikan dalam dua cara: jalur satelit dan jalur kabel optik di dasar laut. Untuk dapat mengirim, Anda perlu memastikan bahwa setiap ponsel  memiliki IP. Alamat IP adalah alamat tujuan pengiriman data dari server Google, dan jika pengiriman data gagal, pemberitahuan akan dikirim dan dikembalikan.
