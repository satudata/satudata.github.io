3. cara menghubungkan modem dan mikrotik selain dengan dhcp client


Cara menghubungkan modem dan mikrotik secara manual :

1. Konfigurasi IP Address Ether 1

Klik Menu IP > Tanda + , isi kolom Address dengan IP ISP yg diberikan sesuai sekolah/ruangan dan isi Kolom Interfaces dengan ether1

2. Konfigurasi DNS

Klik Menu IP > DNS, lalu isi kolom servers dengan IP DNS yang diberikan atau jika tidak ada isi saja dengan IP DNS Google 8.8.8.8 dan 8.8.4.4 dan tandai Allow Remote Request dengan memberi  tanda centang

3. Konfigurasi IP Route

Klik Menu IP > Route > Klik tanda +, isi kolom Dst. Address dengan 0.0.0.0/0 dan kolom Gateway dengan IP Gateway yang diberikan ISP.

Untuk mengecek apakah mikrotik sudah terhubung dengan internet atau belum, caranya dengan :

Klik Menu > New Terminal lalu tuliskan script # ping bluespedia.xyz. dan jika sudah terhubung akan  ditampilkan teks Running berupa Nomor SEQ, Nomor Host, Size, TTL, Time dan status.

Pembahasan

Modem merupakan singkatan Modulator Demodulator. Modulator adalah bagian yang mengubah sinyal informasi ke dalam sinyal pembawa dan siap untuk dikirimkan.Demodulator adalah bagian yang memisahkan sinyal informasi yang berisi data atau pesan dari sinyal pembawa yang diterima sehingga informasi dapat diterima dengan baik.Cara kerja modem yaitu ketika modem menerima data dari luar  berupa sinyal analog, maka modem mengubahnya menjadi sinyal digital supaya dapat diproses lebih lanjut oleh komputer dan dikirimkan melalui beberapa media telekomunikasi seperti telepon dan radio, setelah di modem tujuan, sinyal analog tersebut diubah menjadi sinyal digital kembali dan dikirimkan ke komputer.

14. Sebutkan kelebihan setting ip address dengan DHCP dibanding setting ip static?


Jawaban:

Kemudahan AdministrasiPenghematan WaktuFleksibilitasPengelolaan Pengguna yang MudahMenghindari Konflik IP

Penjelasan:

Berikut adalah beberapa kelebihan setting IP address dengan DHCP (Dynamic Host Configuration Protocol) dibandingkan dengan setting IP static:

Kemudahan Administrasi: DHCP memungkinkan administrasi jaringan yang lebih mudah. Dengan DHCP, pengaturan dan distribusi IP address secara otomatis ditangani oleh server DHCP. Ini mengurangi kerumitan dalam mengelola dan mengkonfigurasi IP address secara manual pada setiap perangkat di jaringan.Penghematan Waktu: Dengan menggunakan DHCP, pengguna atau administrator tidak perlu mengonfigurasi IP address secara manual pada setiap perangkat. Pengalokasian IP address dilakukan secara otomatis oleh server DHCP, sehingga menghemat waktu yang diperlukan untuk mengatur setiap perangkat secara individual.Fleksibilitas: DHCP memungkinkan pengalokasian IP address secara dinamis. Ini berarti bahwa perangkat dapat menerima IP address yang berbeda setiap kali mereka terhubung ke jaringan. Hal ini memudahkan penggunaan perangkat di berbagai lokasi atau jaringan yang berbeda tanpa perlu mengubah pengaturan IP secara manual.Pengelolaan Pengguna yang Mudah: Dalam lingkungan jaringan yang besar, pengelolaan perangkat dan pengguna menjadi lebih mudah dengan DHCP. Administrator dapat dengan mudah mengendalikan dan memonitor IP address yang diberikan kepada setiap perangkat yang terhubung ke jaringan melalui server DHCP sentral.Menghindari Konflik IP: Dengan menggunakan DHCP, risiko terjadinya konflik IP address dapat dikurangi. Server DHCP memastikan bahwa setiap perangkat yang terhubung ke jaringan menerima IP address yang unik, sehingga menghindari konflik IP yang dapat menyebabkan gangguan pada komunikasi dan konektivitas jaringan.

Meskipun DHCP memiliki kelebihan-kelebihan ini, terdapat juga beberapa situasi di mana penggunaan IP address static (tetap) lebih disukai, seperti ketika diperlukan kontrol penuh terhadap alamat IP yang digunakan oleh perangkat, atau ketika perangkat tersebut berfungsi sebagai server yang perlu diakses secara konsisten menggunakan alamat IP yang tetap.

18. ketika kita selesai setting mikrotik 1 lalu kita lanjut mikrotik ke 2 setelah itu kenapa setingan mikrotik 1 masih ada pada setingan mikrotik ke 2​


Penjelasan:

Jika kmu melakukan konfigurasi pada dua perangkat MikroTik yang terpisah, pengaturan pada satu perangkat tidak akan secara otomatis muncul pada perangkat MikroTik yang lain. Meskipun pengaturan di kedua perangkat dapat diatur agar sama atau mirip, namun secara default tidak ada hubungan antara dua perangkat MikroTik yang terpisah.

Mungkin ada beberapa alasan mengapa kmu merasa bahwa pengaturan pada MikroTik pertama muncul pada MikroTik kedua. Salah satu kemungkinan adalah bahwa kmu telah melakukan kloning atau mengimpor konfigurasi MikroTik pertama ke MikroTik kedua, yang mengakibatkan pengaturan pada kedua perangkat menjadi identik. Namun, tanpa tindakan seperti itu, kedua perangkat MikroTik akan memiliki pengaturan yang terpisah dan independen satu sama lain.

20. jelaskan cara setting wifi hotspot mikrotik


Langkah pertama, kita diminta untuk menentukan interface mana Hotspot akan diaktifkan. Pada kasus kali ini, Hotspot diaktifkan pada wlan1, dimana wlan1 sudah kita set sebagai access point (ap-bridge). Selanjutnya klik Next.

Jika di interface wlan1 sudah terdapat IP, maka pada langkah kedua ini, secara otomatis terisi IP Address yang ada di wlan1. Tetapi jika belum terpasang IP, maka kita bisa menentukan IP nya di langkah ini. Kemudian Klik Next.

Langkah ketiga, tentukan range IP Address yang akan diberikan ke user (DHCP Server). Secara default, router otomatis memberikan range IP sesuai dengan prefix/subnet IP yang ada di interface. Tetapi kita bisa merubahnya jika dibutuhkan. Lalu klik Next.

Langkah selanjutnya, menentukan SSL Certificate jika kita akan menggunakan HTTPS untuk halaman loginnya. Tetapi jika kita tidak memiliki sertifikat SSL, kita pilihl none, kemudian klik Next

Jika diperlukan SMTP Server khusus untuk server hotspot bisa ditentukan, sehingga setiap request SMTP client diredirect ke SMTP yang kita tentukan. Karena tidak disediakan smtp server, IP 0.0.0.0 kami biarkan default. Kemudian klik Next.

Di langkah ini, kita meentukan alamat DNS Server. Anda bisa isi dengan DNS yang diberikan oleh ISP atau dengan open DNS. Sebagai contoh, kita menggunakan DNS Server Google. Lalu klik Next.

Selanjutnya kita diminta memasukkan nama DNS untuk local hotspot server. Jika diisikan, nantinya setiap user yang belum melakukan login dan akan akses ke internet, maka browser akan dibelokkan ke halaman login ini. Disini DNS name sebaiknya menggunakan format FQDN yang benar. Jika tidak diisikan maka di halaman login akan menggunakan url IP address dari wlan1. Pada kasus ini, nama DNS-nya diisi "hotspot.mikrotik.co.id". Lalu klik Next.

Langkah terakhir, tentukan username dan pasword untuk login ke jaringan hotspot Anda. Ini adalah username yang akan kita gunakan untuk mencoba jaringan hotspot kita.

Sampai pada langkah ini, jika di klik Next maka akan muncul pesan yang menyatakan bahwa setting Hotspot telah selesai.

22. Jika wlan dan ether2 di-bridge, maka interface manakah yang disetting IP dan setting DHCP server?.


Jawaban:

Settingan IP dan DHCP Server berada pada Bridge Connection

Penjelasan:

Pada kasus permasalahan, diminta WLAN dan Port Ether2 di bridge, konsep bridge sendiri yaitu dapat menghuungkan jaringan/port yang berbeda dan menjadikannya dalam satu jalur koneksi, dalam hal ini WLAN dan Ether2 menjadi 1 koneksi. Maka dari itu, router akan membuat interface baru dibawah bridge menu, contohnya pada mikrotik, ketika kita ingin menjadikannya bridge, maka kita akan membuat interface bridge terlebih dahulu, kemudian mengisikan port yang ingin disatukan. Agar lebih mudahnya, berikut ini merupakan langkah-langkahnya (penulsi menggunakan router Mikrotik RouterOS.

1. Masuk pada menu Bridge, lalu klik tanda (+), kemudian isikan nama interface bridge, setelah itu klik ok

2. Masuk ke tab Ports, lalu isikan interface WLAN dan Ether2

3. Kemudian masuk ke menu IP > Address List, kemudian klik tanda (+), tambahkan IP yang sesuai dan pilih interface bridge (nama bridge yang dibuat tadi).

4. Kemudian, masuk ke menu IP > DHCP Server > klik tanda (+) > isikan name sesuai dengan keinginan, kemudian pilih interface bridge (nama bridge yang telah dibuat)

5. Maka dalam hal ini, bridge yang telah dibuat telah berisikan IP dan sebagai DHCP server

Kesimpulan,

Dari pertanyaan yang diberikan, maka WLAN dan Ether2 di bridge, maka interface yang diberikan IP dan setting DHCP Server adalah interface Bridge.

29. bagaimana cara setting dhcp server?


1. Untuk memulai settingan, klik menu IP, kemudian pilih DHCP Server. Nanti akan muncul jendela DHCP server, kemudian klik DHCP Setup.

2. Selanjutnya kita diperintahkan untuk memilih interface mana yang akan menjalankan DHCP server. Pada contoh ini saya memilih interface yang terhubung ke jaringan local, yaitu “ether2-LAN”. Kemudian klik next.

3. Pada bagian ini kita akan isikan network ID dari IP address jaringan local. Secara default akan terisi menyesuaikan IP address yang digunakan. Klik next.

4. Kemudian isikan IP address yang nantinya akan menjadi default gateway pada sisi klien. Secara default akan terisi IP address yang digunakan mikrotik. Klik next.

5. Selanjutnya kita akan menentukan range IP address yang bisa digunakan oleh klien nantinya.

6. Secara default akan terisi meyesuaikan jumlah host valid dari IP address yang digunakan. Jika terasa terlalu banyak, atau kita ingin menyesuaikannya sendiri, kita bisa mengubah range tersebut. Jika sudah ditentukan range IP address yang akan digunakan, klik next

7. IP DNS yang kita masukan pada bagian ini, nantinya akan menjadi DNS yang digunakan oleh klien. Misal pada contoh ini, saya menggunakan DNS 8.8.8.8. Jika sudah klik next.

8. Ini merupakan wizard terakhir. Pada bagian ini akan menentukan berapa lama waktu peminjaman IP address yang boleh dilakukan oleh setiap klien. Secara otomatis sudah terisi. Kamu bisa menyesuaikannya dengan keinginan kamu. Jika sudah klik next.

9. “Setup has completed successfully”. Klik OK saja. Ini adalah pesan yang menandakan bahwa setting DHCP server telah berhasil. Dan pada table DHCP server juga akan muncul secara otomatis informasi tentang DHCP server yang baru dibuat tadi

Oke, setting DHCP server telah selesai. Sekarang adalah saatnya pengetesan pada sisi klien. Pada sisi klien kita hanya perlu mengkoneksikan secara fisik saja (baik via wireless atau kabel). Jangan berikan settingan apapun pada pengaturan IP address, biarkan semuanya kosong. Karena nanti IP address dan parameter lainnya akan terisi dengan sendirinya.

6. Setelah membuat vlan selanjutnya beri tanda pada masing-masing vlan tersebut, caranya sama seperti memberi tanda pada ethernet.
caranya klik vlan mana yang akan diberi tanda kemudian klik kotak kuning diatas (comment), hingga muncul kotak seperti gambar dibawah kemudian ketik pada kotak tersebut untuk memberikan tanda setelah itu "OK"
7. Sesuaikan tanda pada vlan tersebut akan mengarah ke mana sesuai tujuan yang sudah anda rencanakan
8. Beri IP pada masing-masing vlan agar dapat terhubung dengan perangkat yang akan terhubung pada vlan yang kita buat.
caranya klik menu "IP" => "Addresses"
9. Setelah muncul menu seperti dibawah ini selanjutnya tinggal kita buat IP untuk masing-masing vlan.
caranya klik "add" (+) setelah itu buat IP untuk vlan dan arahkan Interface ke vlan yang sudah kita buat sebelumnya.
kemudian "Apply" => "OK"
10. Lakukan cara seperti diatas untuk membuat IP vlan-vlan selanjutnya, dengan IP Address yang berbeda sesuai rencana / topologi yang sudah anda buat dan arahkan Interface ke vlan-vlan selanjutnya
